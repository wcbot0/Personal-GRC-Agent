# Personal GRC Agent v0.1.0

**Release date:** 2026-06-13

Initial public release of **Personal GRC Agent (PGA)** — a local-first, draft-by-default GRC copilot you run on your own machine.

## Highlights

- **Draft-by-default governance** — agents read and draft freely; assigning humans, publishing policies, or writing to external systems requires an approved Change Proposal Object (CPO)
- **Hash-chained audit trail** — every governed action is logged and verifiable with `spa audit verify`
- **Nine verifier-gated skills** — meeting synthesis, tickets, policy redlines, control crosswalks, evidence packs, daily briefs, risk assessments, repo security reviews, and vendor questionnaires
- **Multi-framework crosswalks** — CSF 2.0, SOC 2, ISO 27001, ISO 27018, ISO 42001, and NIST 800-53 mappings
- **Five runtime adapters** — Cursor, Claude Code, Hermes, ChatGPT, and OpenClaw via `spa init --runtime`

## Quick start

```bash
git clone https://github.com/wcbot0/Personal-GRC-Agent.git
cd Personal-GRC-Agent
./bootstrap.sh
source .venv/bin/activate
spa ingest evals/fixtures/meeting_sample.md
```

See [Getting Started](https://github.com/wcbot0/Personal-GRC-Agent/blob/main/docs/getting-started.md) for the full walkthrough.

## What's included

### CLI (`spa`)

| Command | Purpose |
|---------|---------|
| `spa ingest <file>` | Index memory + auto-pipeline for meetings |
| `spa run-skill <name>` | Run a drafting skill with verifiers |
| `spa mcp serve` | Governed MCP server for chat clients |
| `spa proposals list` | Review pending CPOs |
| `spa audit verify` | Validate hash chain integrity |
| `spa brain add <pack>` | Install optional brain packs (ISO 42001, NIST AI RMF) |
| `spa cloud scan` | Read-only cloud posture checks |

### Skills

`meeting-synth` · `ticket-draft` · `policy-redline` · `csf-crosswalk` · `evidence-pack` · `daily-brief` · `risk-analyst` · `repo-security-review` · `questionnaire`

### Connectors (safe defaults)

- Ticket and GRC providers default to **file-only** mode (`TICKET_PROVIDER=none`)
- Optional Linear live writes behind CPO approval (disabled by default)
- Vendor MCP templates shipped as `.disabled` stubs

## Requirements

- Python 3.11+
- Docker (Qdrant + local embeddings via HuggingFace TEI)
- Git

Optional: LLM API key for richer skill output (heuristic fallback works without one)
